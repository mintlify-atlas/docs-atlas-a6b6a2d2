Guías AESIA 4.0 está construida sobre **Next.js 16 con App Router**, siguiendo una separación clara entre rutas (`app/`), componentes reutilizables (`components/`), lógica de negocio y datos (`lib/`) y recursos estáticos (`public/`). Esta organización permite escalar las 16 guías técnicas y los módulos de juego de forma independiente, manteniendo la coherencia visual y la tipología estricta con TypeScript 5.

## Rutas principales

La aplicación expone las siguientes rutas a través del App Router:

| Ruta | Descripción |
|------|-------------|
| `/` | Página principal con `HeroSection` y catálogo de guías |
| `/guides/[id]` | Vista de contenido de una guía concreta |
| `/guides/[id]/flashcards` | Módulo de flashcards de la guía |
| `/guides/[id]/quiz` | Cuestionario interactivo de la guía |
| `/guides/[id]/glossary` | Glosario de términos de la guía |
| `/escape-room` | Juego de Escape Room temático |
| `/pasapalabra` | Juego Rosco RIA (Pasapalabra) |
| `/aventura` | Juego de rol narrativo IA Aventura |

## Capa de datos

Todos los datos de contenido residen en `lib/data/`. Los tipos TypeScript se exportan desde `lib/data/index.ts` y son compartidos por todos los módulos:

| Interfaz | Descripción |
|----------|-------------|
| `Guide` | Estructura completa de una guía técnica |
| `Flashcard` | Tarjeta de memoria con pregunta y respuesta |
| `QuizQuestion` | Pregunta de opción múltiple con explicación |
| `GlossaryTerm` | Término del glosario con definición |
| `ContentSection` | Sección de contenido de una guía |

La función `getGuides(lang)` de `lib/data/index.ts` es el punto de entrada principal: devuelve el array de guías para el idioma solicitado (`es`, `gl`, `ca`, `eu`) con `es` como fallback por defecto.

## Sistema i18n

El soporte multiidioma se implementa con dos capas independientes:

- **`LanguageProvider`** (`components/LanguageProvider.tsx`) — contexto React que expone el idioma activo y el diccionario de cadenas de interfaz. Envuelve toda la aplicación desde `app/layout.tsx`.
- **`lib/data/`** — archivos de datos separados por idioma (`es.ts`, `gl.ts`, `ca.ts`, `eu.ts`). El inglés (`en.ts`) está en desarrollo.
- **`lib/i18n/`** — diccionarios de traducciones de los textos de la interfaz de usuario.

## Sistema de temas

El soporte claro/oscuro se gestiona mediante:

- **`ThemeProvider`** (`components/ThemeProvider.tsx`) — contexto que almacena el tema activo en `localStorage` y lo aplica como clase CSS al `<html>`.
- **Script de inicialización inline** en `app/layout.tsx` — se ejecuta antes del primer render para leer `localStorage` y aplicar la clase de tema, evitando el parpadeo (flash) en la carga inicial.
- **`ThemeToggle`** — componente de alternancia integrado en la `Navbar`.
